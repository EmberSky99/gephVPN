## 🛠 技术实现与安全性

### 为什么我们敢说“不怕封锁”？

大多数 VPN 被封锁是因为服务器 IP 被直接拉黑。迷雾通采用**双跳架构**：

1. **桥接节点 (Bridge)：** 用户通过不断轮换、动态分配的桥接节点接入。
2. **出口节点 (Exit)：** 最终访问流量从出口流出。
审查者或许能封锁出口，但由于我们的协议采用了不同用户分配不同桥接的策略，几乎无法全面枚举和封锁。自 2019 年以来，这种架构已多次证明了其卓越的抗封锁能力。

### 什么是盲签名 (Blind Signatures)？

这是迷雾通匿名性的核心。当您连接时，客户端会提交一个匿名证明。服务器可以确认“这是一个合法的免费/Plus用户”，但它**无法知道**这个用户具体是谁。这种设计在物理层面上切断了用户身份与浏览记录的联系。
