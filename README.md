## 🛠 技术实现：为什么我们不怕封锁？

### 1. 双跳架构 (Two-hop Architecture)

大多数 VPN 容易被封锁是因为出口服务器 IP 被直接识别。迷雾通采用**双跳架构**：

* **桥接节点 (Bridge)：** 用户通过不断轮换、动态分配的桥接节点接入。
* **出口节点 (Exit)：** 流量最终从出口流出。
审查者或许能封锁出口，但无法全面枚举和封锁不断变化的桥接点。自 2019 年以来，这种架构已成功抵御了多次自动化封锁。

### 2. 盲签名 (Blind Signatures)

迷雾通实现了真正的匿名。当您连接时，客户端会提交一个匿名证明。服务器只能确认“这是一个合法的用户”，但它**无法识别**你是谁。这意味着你的身份与你的上网活动是完全物理隔离的。
